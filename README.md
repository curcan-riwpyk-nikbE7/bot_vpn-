# VPN Telegram Bot

Профессиональный Telegram-бот для продажи VPN-подписок на Python (aiogram 3 + SQLite). Всё настраивается через админ-панель прямо в боте — серверы, тарифы, ключи. Редактировать код для настройки не нужно.

## 🚀 Возможности

- 📋 Динамические тарифы (настраиваются через админ-панель)
- 🔑 Автоматическая генерация VPN-ключей (WireGuard / V2Ray-VLESS / OpenVPN)
- 💳 Платежи: встроенный демо-режим + поддержка Telegram Payments (provider token)
- 📊 Полная административная панель с inline-кнопками
- 🖥️ Управление VPN-серверами через бот (добавление / лимиты / удаление)
- ⚖️ Автоматическая балансировка нагрузки между серверами
- 💰 Управление тарифами и ценами через бот
- 🔑 Ручное создание ключей для любого пользователя
- 💾 SQLite (через aiosqlite) для хранения данных
- 🔒 Секреты только в `.env`, база и `.env` исключены из git

## 📋 Требования

- Python 3.10+
- Telegram Bot Token (получить у [@BotFather](https://t.me/botfather))

## 🔧 Установка

```bash
git clone <repo-url>
cd vpn-telegram-bot
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Отредактируйте `.env`:

```env
BOT_TOKEN=your_telegram_bot_token_here
ADMIN_ID=123456789                 # ваш id, можно несколько через запятую
PAYMENT_PROVIDER_TOKEN=            # пусто = демо-режим оплаты
CURRENCY=RUB
DATABASE_PATH=vpn_bot.db
```

### Получение BOT_TOKEN
1. Откройте [@BotFather](https://t.me/botfather), команда `/newbot`, следуйте инструкциям, скопируйте токен.

### Получение ADMIN_ID
1. Откройте [@userinfobot](https://t.me/userinfobot), отправьте любое сообщение, скопируйте ваш `Id`.

## 📱 Команды бота

- `/start` — главное меню (купить VPN, мои ключи, помощь)
- `/admin` — административная панель (только для админов из `ADMIN_ID`)

## 🛠️ Административная панель

Откройте `/admin`. Доступные разделы:

### 📊 Статистика
Пользователи, активные ключи, выручка, число активных серверов и тарифов.

### 🖥️ Управление серверами
- **Добавить сервер**: название → адрес → порт → протокол (WireGuard / V2Ray / OpenVPN) → публичный ключ сервера (для WG/V2Ray) → лимит подключений
- **Просмотр серверов**: список с нагрузкой в % и статусом
- **Изменить лимит**: максимум подключений на сервер
- **Удалить сервер**: мягкое отключение (is_active = 0)

### ⚖️ Балансировка нагрузки
Бот автоматически выбирает наименее загруженный активный сервер со свободными слотами. Если свободных серверов нет — пользователь получает понятное сообщение.

### 💰 Управление тарифами
- **Добавить тариф**: название → дни → цена → описание
- **Просмотр / удаление тарифов** (мягкое отключение)

### 🔑 Ручное создание ключей
Создание ключа любому пользователю по его Telegram ID: выбор сервера и тарифа, автогенерация и отправка конфигурации этому пользователю.

## 🧩 Протоколы и выдача ключей

- **WireGuard** — генерируется клиентский `.conf` (с уникальной парой ключей Curve25519), отправляется текстом и файлом для импорта.
- **V2Ray / VLESS** — генерируется ссылка `vless://...` для клиентов (v2rayNG, Nekoray и др.).
- **OpenVPN** — генерируются логин/пароль и минимальный `.ovpn`-профиль.

> Конфигурации формируются ботом и сразу выдаются клиенту. Для боевого VPN-сервера привяжите выдачу к реальному серверу (push публичного ключа через `wg`, OpenVPN management interface и т.п.) в функции `_deliver_key` (`bot.py`) и `vpn_generator.generate`.

## 💳 Платежи

- **Демо-режим** (по умолчанию, если `PAYMENT_PROVIDER_TOKEN` пуст): оплата подтверждается мгновенно, ключ выдаётся сразу — удобно для тестов.
- **Telegram Payments**: задайте `PAYMENT_PROVIDER_TOKEN` (получается у [@BotFather](https://t.me/botfather) → Payments). Бот отправит инвойс, обработает `pre_checkout` и `successful_payment`, после оплаты выдаст ключ.

## 🗄️ Схема БД

## 🔒 Безопасность

- Никогда не публикуйте `.env` (он в `.gitignore`).
- База `*.db` также исключена из git.
- Доступ к админ-функциям ограничен списком `ADMIN_ID`.
- Регулярно обновляйте зависимости.

## 🐛 Устранение проблем

**Бот не запускается** — проверьте `BOT_TOKEN` и `ADMIN_ID` в `.env`, установлены ли зависимости, посмотрите логи.

**Ключи не работают** — для боевого использования нужно привязать выдачу к реально настроенному VPN-серверу (см. раздел «Протоколы»). Проверьте адрес/порт сервера в админ-панели и что порт открыт.
