# VPN Telegram Bot

Профессиональный Telegram-бот для продажи VPN-подписок на Python (aiogram 3 + SQLite). Всё настраивается через админ-панель прямо в боте — серверы, тарифы, ключи. Редактировать код для настройки не нужно.

## 🚀 Возможности

- 📋 Динамические тарифы (настраиваются через админ-панель)
- 🔑 Автоматическая генерация VPN-ключей (WireGuard / V2Ray-VLESS / OpenVPN)
- 💳 Платежи: встроенный демо-режим + поддержка Telegram Payments (provider token)
- 📊 Полная административная панель с inline-кнопками
- 🖥️ Управление VPN-серверами через бот (добавление / лимиты / удаление)
- ⚖️ Автоматическая балансировка нагрузки между серверами
- 💰 Управление тарифами и ценами через бот
- 🔑 Ручное создание ключей для любого пользователя
- 🚀 Скрипт `install_vpn.sh` для установки WireGuard-сервера в одну команду
- 🤖 Автоматическая выдача рабочих ключей: бот сам добавляет peer на сервер по SSH
- 🛡️ Управление панелями 3X-UI прямо из бота: создание VLESS-клиента и выдача `vless://`-ссылки через API
- 💾 SQLite (через aiosqlite) для хранения данных
- 🔒 Секреты только в `.env`, база и `.env` исключены из git

## 📋 Требования

- Python 3.10+
- Telegram Bot Token (получить у [@BotFather](https://t.me/botfather))

## 🔧 Установка

```bash
git clone <repo-url>
cd vpn-telegram-bot
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Отредактируйте `.env`:

```env
BOT_TOKEN=your_telegram_bot_token_here
ADMIN_ID=123456789                 # ваш id, можно несколько через запятую
PAYMENT_PROVIDER_TOKEN=            # пусто = демо-режим оплаты
CURRENCY=RUB
DATABASE_PATH=vpn_bot.db
```

### Получение BOT_TOKEN
1. Откройте [@BotFather](https://t.me/botfather), команда `/newbot`, следуйте инструкциям, скопируйте токен.

### Получение ADMIN_ID
1. Откройте [@userinfobot](https://t.me/userinfobot), отправьте любое сообщение, скопируйте ваш `Id`.

## 📱 Команды бота

- `/start` — главное меню (купить VPN, мои ключи, помощь)
- `/admin` — административная панель (только для админов из `ADMIN_ID`)

## 🛠️ Административная панель

Откройте `/admin`. Доступные разделы:

### 📊 Статистика
Пользователи, активные ключи, выручка, число активных серверов и тарифов.

### 🖥️ Управление серверами
- **Добавить сервер**: название → адрес → порт → протокол (WireGuard / V2Ray / OpenVPN) → публичный ключ сервера (для WG/V2Ray) → лимит подключений
- **Просмотр серверов**: список с нагрузкой в % и статусом
- **Изменить лимит**: максимум подключений на сервер
- **Удалить сервер**: мягкое отключение (is_active = 0)

### ⚖️ Балансировка нагрузки
Бот автоматически выбирает наименее загруженный активный сервер со свободными слотами. Если свободных серверов нет — пользователь получает понятное сообщение.

### 💰 Управление тарифами
- **Добавить тариф**: название → дни → цена → описание
- **Просмотр / удаление тарифов** (мягкое отключение)

### 🔑 Ручное создание ключей
Создание ключа любому пользователю по его Telegram ID: выбор сервера и тарифа, автогенерация и отправка конфигурации этому пользователю.

## 🚀 Установка VPN-сервера (WireGuard) одной командой

На чистом сервере с Ubuntu/Debian (нужен root):

```bash
sudo bash install_vpn.sh
```

Скрипт установит WireGuard, сгенерирует ключи сервера, поднимет интерфейс `wg0`, включит форвардинг/NAT/фаервол и автозапуск. В конце он напечатает данные для добавления сервера в боте:

```
Протокол:        WireGuard
Адрес (host):    <публичный IP>
Порт (port):     51820
Публичный ключ:  <server public key>
```

Параметры можно переопределить переменными окружения: `WG_PORT`, `WG_ADDR`, `WG_IF`, `WG_DNS`.

## 🤖 Автоматическая выдача ключей (WireGuard)

Чтобы выданные ключи **сразу работали**, бот может сам регистрировать клиента (peer) на сервере по SSH при покупке. Включается в `.env`:

```env
WG_AUTO_PROVISION=true
WG_SSH_USER=root
WG_SSH_PORT=22
WG_SSH_KEY=/path/to/id_ed25519     # ИЛИ WG_SSH_PASSWORD=...
WG_INTERFACE=wg0                    # как в install_vpn.sh
WG_SUBNET=10.66.66.0/24
WG_DNS=1.1.1.1, 8.8.8.8
```

При выдаче ключа бот подключается к серверу (по его `host` из админ-панели), выделяет свободный IP в туннеле, выполняет `wg set ... + wg-quick save` и отдаёт пользователю готовый `.conf`. Если сервер недоступен — бот логирует ошибку и отдаёт офлайн-конфиг (без регистрации peer).

## 🛡️ Панель 3X-UI (VLESS) прямо в боте

Бот умеет управлять панелями [3X-UI](https://github.com/MHSanaei/3x-ui): при покупке он сам создаёт клиента на нужном inbound через API панели и выдаёт пользователю готовую ссылку `vless://...`.

Как подключить:

1. В админ-панели нажмите **«➕ Добавить 3X-UI панель»** (`/admin` → 🖥️ Серверы).
2. Введите по шагам: название → URL панели (например `http://1.2.3.4:54321` или `https://panel.site/secret`) → логин → пароль → ID inbound → адрес для клиентов (или «-», чтобы взять из URL) → лимит подключений.
3. Бот сразу проверит подключение к панели (логин + наличие inbound) и сообщит результат.

После этого панель работает как обычный «сервер»: участвует в балансировке нагрузки, а при покупке клиент создаётся на панели автоматически. Поддерживаются inbound'ы с `reality`, `tls` и без шифрования, транспорты TCP/WS/gRPC/HTTP — ссылка собирается из настроек inbound. Глобальные параметры — `XUI_FLOW` и `XUI_VERIFY_SSL` (см. `.env.example`).

## 🧩 Протоколы и выдача ключей

- **WireGuard** — генерируется клиентский `.conf` (с уникальной парой ключей Curve25519); при `WG_AUTO_PROVISION=true` peer добавляется на сервер автоматически.
- **3X-UI / VLESS** — клиент создаётся на панели 3X-UI через API, пользователь получает рабочую ссылку `vless://...`.
- **V2Ray / VLESS (без панели)** — генерируется ссылка `vless://...` для клиентов (v2rayNG, Nekoray и др.).
- **OpenVPN** — генерируются логин/пароль и минимальный `.ovpn`-профиль.

> Авто-провижининг реализован для WireGuard (по SSH) и 3X-UI (по API панели). Для офлайн V2Ray/OpenVPN выдача формирует конфиг, привязка к серверу — точка расширения в `_deliver_key` (`bot.py`).

## 💳 Платежи

- **Демо-режим** (по умолчанию, если `PAYMENT_PROVIDER_TOKEN` пуст): оплата подтверждается мгновенно, ключ выдаётся сразу — удобно для тестов.
- **Telegram Payments**: задайте `PAYMENT_PROVIDER_TOKEN` (получается у [@BotFather](https://t.me/botfather) → Payments). Бот отправит инвойс, обработает `pre_checkout` и `successful_payment`, после оплаты выдаст ключ.

## 🔒 Безопасность

- Никогда не публикуйте `.env` (он в `.gitignore`).
- База `*.db` также исключена из git.
- Доступ к админ-функциям ограничен списком `ADMIN_ID`.
- Регулярно обновляйте зависимости.

## 🐛 Устранение проблем

**Бот не запускается** — проверьте `BOT_TOKEN` и `ADMIN_ID` в `.env`, установлены ли зависимости, посмотрите логи.

**Ключи не работают** — для боевого использования нужно привязать выдачу к реально настроенному VPN-серверу (см. раздел «Протоколы»). Проверьте адрес/порт сервера в админ-панели и что порт открыт.
